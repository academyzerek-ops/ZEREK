# B1 — Владелец-сдатчик (пассивный)

**Владелец салона/студии сдаёт места мастерам за фиксированную аренду или процент от их выручки.**

## Описание

Владелец = администратор инфраструктуры, не мастер. Не делает услуги клиентам напрямую. Получает доход от:
- (вариант α) фиксированной аренды места × N мастеров
- (вариант β) % с выручки каждого мастера (commission-in)
- комбинаций обоих

Несёт расходы: аренда помещения, ремонт, общий маркетинг бренда салона, ресепшен (если есть), ЖКУ, мелочёвка.

## Формула (концепция)

```
N_masters       = количество мастеров на точке (параметр)
master_revenue  = средняя выручка одного мастера в месяц (mature)
revenue_owner   = N_masters × master_revenue × commission_in  (вариант β)
                  ИЛИ  N_masters × rent_per_master            (вариант α)
                  ИЛИ  гибрид (фикс + %)

opex_owner      = rent_full_premise + marketing_brand + reception_fot + utilities + other_opex
materials_owner = расходники общего пользования (стерилизация, дезинфекция, бумажные принадлежности)
tax             = revenue_owner × tax_rate (TOO 3% упрощёнка обычно)
pocket_owner    = revenue_owner − opex − materials_owner − tax
```

## Используется в

- **BEAUTY салон PASSIVE** (зонтичная beauty-ниша, режим 1 из 2)
- **BARBER SHOP_RENT_OUT** — владелец барбершопа сдаёт кресла (4-6 мастеров)
- (потенциально) COSMETOLOGY салон без врачей-сотрудников

## Связь с мастерами

Мастера в B1-салоне работают по архетипу A2 (commission_out на уровне мастера = commission_in на уровне владельца). Если ZEREK хочет показать обе стороны — расчёт B1 + расчёт A2 для одного и того же салона.

## Особенности

- Владелец **не делает услуги**. Если хочет ещё и стричь сам — это уже гибрид B1+A1, либо B2.
- N_masters может быть параметром выбора (сколько кресел сдаёт).
- Налог: обычно TOO упрощёнка (3%), поскольку оборот = N × несколько М/мес, превышает ИП-СЗ предел быстро.

## Что НЕ надо ставить в B1

- ФОТ мастеров — мастера за commission, не за оклад. Если мастера на ФОТ — это **B2**.
- Свои services (массаж владельца клиентам) — это нарушает «pure passive».
