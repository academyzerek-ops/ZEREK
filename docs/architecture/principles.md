# Принципы (общие правила всех ниш)

Применяются ко всем нишам ZEREK. Если возникает противоречие в спеке ниши — побеждает этот файл.

## 1. Целевая аудитория

ZEREK считает **экономику бизнеса/самозанятости**. Не зарплату работника по найму.

**Найм за оклад исключается** из всех ниш. Если бизнес-модель = "пойти в найм за зарплату X" — это не ниша ZEREK.

## 2. Ниша = реалия рынка

Не подгонять под пресловутые «50 ниш». Каждая ниша содержит **столько форматов, сколько в реальной казахстанской практике** — не больше, не меньше.

Пример: у MANICURE 4 формата (HOME / STUDIO simple+nice / SALON_RENT std+prem / MALL_SOLO), потому что это и есть весь спектр casa-индустрии. У SUGARING — только 3 (HOME / STUDIO simple+nice), потому что в КЗ этого сегмента нет ни в салон-аренде, ни в ТЦ.

## 3. Три локации универсально

Спальник / центр / ТЦ — три локации, применяющиеся ко всем форматам со своей точкой. Аренды:
- спальник: low rent, low traffic
- центр: mid rent, mid-high traffic
- ТЦ открытая стойка: 30-70K (med ~50K), высокий traffic, низкая средняя выручка
- ТЦ бутик: 5-10K/м², закрытое помещение, выше средний чек

## 4. Локация ≠ ниша

Локация — это **параметр выбора** (loc_type / level), не отдельная запись в реестре ниш. Один format_id может быть валидным в нескольких локациях.

## 5. Разделение мастер vs владелец

Если в нише существуют обе модели — внутри ниши **и расчёт мастера, и расчёт владельца**.

Пример: BARBER. Мастер: STUDIO + SALON_RENT. Владелец: SHOP_RENT_OUT + SHOP_EMPLOY. Это **одна ниша** с двумя видами расчёта.

## 6. Зонтичный «BEAUTY салон»

Владелец салона с мульти-мастерным составом (стрижки + маникюр + брови + ...) — это **единая ниша BEAUTY**. Не плодить NAILSALON, BARBERSHOP, BROWBAR, MASSAGESALON и т.п. как отдельные записи.

BEAUTY = 2 режима (PASSIVE/ACTIVE) × 3 локации (suburb/center/mall) = 6 конфигураций. Один format_id с уровнями.

## Якорь для калибровщика

Перед калибровкой ниши:
1. Открыть [categories/](categories/) для категории.
2. Прочитать описание ниши там — формат-список + архетип.
3. Открыть [archetypes/](archetypes/) для архетипа — формула + поля.
4. Если в файле нет ответа — спросить Ноа, она вписывает в файл.
5. После — калибровать через `data/niches/<CODE>_data.yaml` + sync xlsx.
