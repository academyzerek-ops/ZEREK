## 1. Wiki-обзоры (бесплатно, 50+ ниш, цель — сотни)

**Назначение.** Верх воронки. Обзор ниши на 5–10 минут чтения. Помогает потенциальному предпринимателю **выбрать или отвергнуть нишу осознанно** — до того, как он начнёт тратить деньги.

**Что внутри обзора:**
- Суть бизнеса (что это, кому продаём, чем зарабатываем)
- Экономика и маржинальность
- Стартовые вложения по форматам
- Юнит-экономика (чек, частота, средние объёмы)
- Сезонность (если есть) с математикой
- Кадры, команда, типичная структура
- Конкуренция (таблицы 2GIS по 22 городам КЗ)
- Налоги (с учётом особенностей ниши)
- 3 типа кейсов: «Хороший старт», «Типичная ошибка», «На грани»
- Риски (бренд-бизнес vs ремесло, кадровый вопрос, сезонность)
- 5 наставлений (см. `05_QC_METHODOLOGY.md`)
- CTA в конце: пара **QC $10 + FinModel $20**

**Тон обзора.** Тот же, что во всём ZEREK — реалистичный, без надувания. Числа — с источником или диапазоном.

**Технические правила Wiki — строго:**
- ЗАПРЕТ выдумывать рыночные числа («130–180 млрд ₸») — нет в insight/БНС → не писать
- ЗАПРЕТ выдумывать кейсы — либо реальные с YouTube, либо помечать «иллюстративный пример»
- Коэффициенты городов **в текст не выводить** (это внутренняя механика движка, не для клиента)
- Таблицы городов: только конкуренты 2GIS + средний чек + инсайт
- В рисках — обязательно: разница «бренд-бизнес vs ремесло» + блок про качество рабочей силы в КЗ + ЗП по средним КЗ, не по Актобе
- Hero: 80vh, full-width, base64 + blur(2px) + brightness(.35), navigation **под** hero
- Шрифты: Playfair Display + Source Sans 3 + JetBrains Mono
- Мобильные таблицы: 9.5px + nowrap + 5px padding
- Уникальная палитра под нишу

**Где живёт.** GitHub Pages → zerek.cc/wiki/kz/ (для КЗ), позже /wiki/ru/ для России.

## 2. Quick Check — $10 *(LIVE)*

**Назначение.** Tripwire-продукт. Не максимизация маржи, а **генерация доверия и подготовка к апселлу**. Цель — клиент видит за 5 минут расчёта, что у ZEREK в основе реальная математика, и готов платить за более глубокий анализ (FinModel, BizPlan).

**Позиционирование.** Не калькулятор, а **наставник**. За $10 клиент получает:
1. Mature pocket (зрелый месяц М6–М12) — реальные числа после разгона
2. Среднее за первый год с ramp-up — что будет в реальности
3. Предупреждения о рисках в формате 5 наставлений
4. Сценарий 2–3 года вперёд
5. Конкретные рекомендации

Один без другого = неполная картина. Не возвращаемся к этому решению.

**Анкета (Telegram Mini App).** 6–11 шагов в зависимости от ниши:
- Шаги 1–3: общие — город, ниша, формат
- Шаги 4+: динамические по нише (см. `04_NICHES.md`)
- Шаг с маркетингом — обязательная 3-я ось анкеты (Консервативная / Средняя / Агрессивная) (см. `05_QC_METHODOLOGY.md`)

**Отчёт. Mini App** — короткий дашборд (1–2 экрана). **PDF** — полный отчёт уровня консалтинга, скачиваемый через WeasyPrint.

**5 обязательных блоков отчёта** (см. `05_QC_METHODOLOGY.md`): Ramp-up, Сезонность, Маркетинг, Кадры, Резерв на старте.

**Целевая аудитория.** Прошла фильтр (см. `01_VISION.md`): не наёмник за оклад. Только самозанятые на проценте или владельцы своего дела.

**Где живёт.** `products/quick-check.html` (GitHub Pages), API на Railway.

## 3. FinModel — $20

**Назначение.** Главный источник первой выручки. Не апсейл «для галочки», а инструмент, за который клиент готов платить, потому что получает **рабочий Excel** с открытыми параметрами + **PDF-объяснение простым языком**.

**Что отдаём:**
- Excel-финмодель на 12–60 месяцев с защитой формул и открытыми вводными
- P&L, Cash Flow, балансы
- Стресс-тест и анализ чувствительности
- Учёт ramp-up, сезонности, маркетингового бюджета
- Кредитный блок (опционально)
- PDF-объяснение: что значат результаты, что такое ramp-up, как влияет сезонность, что меняется при разных параметрах

**Анкета:** 11 шагов. Из ~40 жёлтых ячеек Excel клиент заполняет ~18 (то, что он реально знает или должен решить); остальные ~22 параметра подставляются автоматически из базы по нише/формату/городу.

**Эндпоинт `/finmodel/defaults/{format_id}`** возвращает бенчмарки → анкета показывает подсказки типа «Средний чек в Актобе: 1 400 ₸».

**Гипотеза:** этот продукт уже воспринимается как серьёзный рабочий инструмент.

## 4. BizPlan + питч — $20

**Назначение.** Премиальный продукт под жёсткие шаблоны: банки, Даму, гранты. Клиент получает документ, **максимально приближенный к готовому формату для подачи**.

**Что отдаём:**
- Бизнес-план по структуре, согласованной с КЗ-институтами (Даму, БРК-Лизинг, банки второго уровня)
- Включает финмодель из FinModel
- Маркетинговый раздел
- Анализ рынка с привязкой к городу/региону
- Раздел рисков
- Резюме проекта на первой странице

**Гипотеза:** премиальный продукт с **высоким риском по качеству исполнения**. Малейшее отклонение от требований банка/фонда обесценивает результат. Поэтому требования к шаблонам — критичны; пишутся под конкретные институции.

## 5. Presentation — питч-дек

**Назначение.** Дополнение к BizPlan для тех, кто идёт питчиться: гранты, инвесторы, банки. Структурированный дек на 10–15 слайдов.

**Цена и формат:** в стадии проработки. Пока — как часть BizPlan-пакета или отдельно.

## Принципы апселла

**Между QC и FinModel.** В отчёте QC — конкретные ограничения: «это разовый снимок, FinModel даст вам 60 месяцев с возможностью менять параметры». Не «купи ещё», а «вот что тебе ещё пригодится».

**Между FinModel и BizPlan.** Если клиент идёт за финансированием — нужен документ под формат банка/Даму. FinModel сама по себе не подаётся, BizPlan — подаётся.

**Между BizPlan и Presentation.** Если клиент идёт на питч — нужна визуальная упаковка. Документ для чтения и слайды для показа — разные жанры.
