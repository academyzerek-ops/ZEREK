**Не работаем над Блоком 2 в активном режиме.** Адиль начнёт делать скелет сам, когда придёт время — а время приходит тогда, когда Блок 1 даёт стабильную месячную выручку и есть деньги на разработчика или на собственное время.

При возврате к теме — поднимать этот документ.

**Не давить, не торопить, не предлагать «давай начнём параллельно».** Расфокусировка убьёт оба блока. Это закрытое решение.

## Видение продукта

ZEREK Блок 2 — **SaaS-приложение, заменяющее аутсорс-бухгалтера** для индивидуального предпринимателя в Казахстане. Не «ещё одна бухгалтерия от банка», а инструмент, который видит **полную картину** бизнеса: банковские счета + наличные + операции + налоги + контрагенты + платёжный календарь.

Главное отличие от банковских бухгалтерий (Kaspi, Halyk, Forte и др.): **банки видят только свои транзакции**. Мы видим всё.

## Целевая аудитория

**Кто:**
- ИП на упрощёнке (УСН)
- 1–10 сотрудников
- B2C или B2B малого масштаба
- Платит сейчас аутсорс-бухгалтеру 20–50K ₸/мес (или забивает на учёт и потом получает штрафы)
- Не разбирается в бухгалтерии и не хочет разбираться

**Кто НЕ ЦА:**
- Крупный бизнес (50+ сотрудников)
- ОУР с НДС, с большой бухгалтерской командой
- Только-только стартующие (для них Блок 1 — Wiki, QC, FM)

## Ключевые фичи (видение, не утверждённый список)

### Базовый слой (must-have для MVP)
- **Загрузка выписок Kaspi и Halyk** — автоматический парсинг, категоризация
- **Голосовой ввод наличных операций** — диктуешь «принял 15 тысяч от Серика за работу», система записывает
- **Автозаполнение деклараций 910.00 и 200.00** — автоматическая сборка отчёта на основе данных
- **Платёжный календарь** — что когда платить (налоги, ОПВ, страховые, поставщикам)
- **AI-уведомления** — «у тебя завтра 910.00, сумма к доплате 18K, готов отправить?»

### Документы голосом
- Счёт на оплату
- Акт выполненных работ
- Счёт-фактура (СФ)
- Доверенность

«Сделай счёт Серикбаю на 250 тысяч за консультацию» → готовый документ, подписан, отправлен в чат.

### Контрагенты
- **Проверка контрагентов по БИН** — внешние сервисы (КГД, статистика)
- **История работы с контрагентом** — сколько за всё время оплачено / получено, надёжность

### Маркетплейс контрагентов (триггер ≥5000 пользователей)
- Каталог проверенных бухгалтеров, юристов, поставщиков, маркетинговых подрядчиков
- Лиды по комиссии или за премиальное размещение

### Управленческий учёт (горизонт после MVP)
- P&L по бизнесу автоматически
- Cash Flow
- Анализ маржи по продуктам / услугам

### Подача отчётности через государственные API (горизонт)
- Прямая подача 910.00 через Smart Bridge / API КГД РК
- Подпись через ЭЦП / eGov Mobile
- **Технически сложно**, нестабильно, UX подписания ЭЦП — отдельная боль. Поэтому только после авторитета и денег.

## Бизнес-модель

**Подписка SaaS** с двумя тарифами (примерные ориентиры, конкретика на момент запуска):

- **Месячная:** 20 000 ₸/мес
- **Годовая со скидкой 30%:** 168 000 ₸/год = 14 000 ₸/мес
- **Первый месяц бесплатно** (trial)

**Целевой ARPU:** 16–18K ₸/мес.

**Замена аутсорс-бухгалтера:** клиент платит сейчас 20–50K ₸/мес бухгалтеру. ZEREK Блок 2 за 14–20K ₸/мес даёт больше функций (управленческий учёт, голос, документы, маркетплейс) → клиенту это **выгодно даже без скидки**.

**Альтернативные модели — рассматриваются, не приняты:**
- Pay-per-use (каждое действие платное) — отвергнуто, уменьшает retention
- Гибрид (базовый бесплатно + платные премиум-фичи) — рассматривается, отложено
- Freemium с лимитом операций — рассматривается, отложено

## Стек разработки — отложен

**Адиль начнёт сам со скелета**, когда появится время и деньги. Ноа поможет ускорить.

**Кандидаты на стек:**
- **Flutter** — полноценное натив-приложение (Android + iOS из одного кода). Лучший UX, выше cost разработки, нужен Flutter-разработчик
- **PWA (Progressive Web App)** — на одном HTML/JS, работает в браузере и устанавливается как приложение. Дешевле, медленнее, ограничения с уведомлениями
- **Telegram Mini App** — продолжение Блока 1, но Mini App имеет жёсткие ограничения для длительной работы и больших данных. Возможно — для лёгких сценариев

**Решение — отложено до момента старта Блока 2.**

## Первый сценарий MVP — отложен

Какую фичу делать первой — это узкое место, от которого зависит окно валидации. Рассматривались варианты:

- **Декларация 910.00 автозаполнение** — закрывает боль раз в полгода, легче валидируется
- **Голосовой учёт наличных** — закрывает ежедневную боль, ярче wow-эффект
- **AI-уведомления о платежах** — низкая стоимость, retention-фокус

Решение — отложено. На момент возврата к Блоку 2: пересмотреть, исходя из того, **на чём Блок 1-аудитория жалуется громче всего к тому моменту**.

## Технические риски и сложности

- **Интеграция с Kaspi и Halyk** — банки не дают полноценные API для всех клиентов. Парсинг выписок (PDF / Excel / CSV) — реалистичный путь, но требует поддержки форматов
- **Интеграция с КГД РК / Smart Bridge** — нестабильная инфраструктура, регулярные изменения, документация фрагментарна
- **ЭЦП и eGov Mobile** — UX подписания ужасный по умолчанию. Нужна работа над тем, чтобы это было легко в нашем приложении
- **Юридические нюансы** — мы не оказываем бухгалтерские услуги, мы даём инструмент для самостоятельного учёта. Это разные правовые статусы. Юрист на этапе запуска — обязателен

## Конкуренция

**Прямые конкуренты в КЗ (на момент написания):**
- **Бухгалтерии банков** (Kaspi Business, Halyk Business, Forte Business) — основной массовый игрок. Сильны интеграцией с банком, слабы тем, что видят только свои транзакции
- **Контур / другие 1С-семейства** — сложные, профессиональные, не для ИП на упрощёнке
- **Аутсорс-бухгалтеры (физлица)** — основная масса ЦА сейчас обслуживается так. Дорого, медленно, ошибки

**Окно — 1–2 года** до того, как Каспи или другой банк выпустит сопоставимый продукт. Используем это окно. После — наша дифференциация это:
- Управленческий учёт (банки этого не дают)
- Голос как ввод (банки этого не делают)
- Маркетплейс контрагентов (банкам это не нужно)
- Связь с Блоком 1 — клиент пришёл из Wiki/Academy → лояльность ZEREK выше, чем к банку

## Связь с Блоком 1 (data flywheel)

**Реальные данные пользователей Блока 2 → улучшение бенчмарков Блока 1.**

Когда у нас есть 5 000 активных пользователей Блока 2 — у нас данные:
- Реальная сезонность по нишам (не оценка из видеоразборов, а факт)
- Средние чеки по городам
- Расходы по статьям
- ФОТ-структуры
- Сезонные кассовые разрывы

Эти данные (анонимизированные) делают Блок 1 (QC, FM, BizPlan, Wiki) **в разы точнее**. И конкурент без своей пользовательской базы не воспроизведёт.

## Что НЕ Блок 2

- Бизнес-аналитика и расчёт идей (это Блок 1)
- Финмодели и бизнес-планы (это Блок 1)
- Обучение и Академия (это Блок 1)
- Налоговая аналитика для крупных компаний (это не наша ЦА)
- Аудиторские услуги (это не наша роль)

## Триггер активации

Возвращаемся к работе над Блоком 2 когда:
- Блок 1 даёт стабильно 1.5–2.5 млн ₸/мес выручки
- Адиль перестал решать ежедневные операционные вопросы Блока 1
- Есть деньги на Flutter-разработчика на 3–6 месяцев работы
- ИЛИ — у Адиля есть свободные 4–6 месяцев на построение скелета самому

**До этого — не открывать тему.**
