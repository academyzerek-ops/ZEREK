## Принцип

Налоговые расчёты в QC / FinModel / BizPlan читают значения только из SoT. Никаких хардкодов в коде, никаких дублей в текстовых файлах. Если в этом документе появилась конкретная цифра — это ошибка дизайна, цифру надо вынести в YAML.

## Структура SoT

В `data/external/kz_tax_constants_2026.yaml`:

- `base_units` — МРП, МЗП, прожиточный минимум, пенсии
- `ip_self_payments` — платежи ИП «за себя» (ОПВ, ОПВР, СО, ВОСМС)
- `ip_minimum_monthly_payment` — минимальный ежемесячный платёж ИП
- `employee_payments` — платежи за сотрудников (для ТОО / ИП с наймом)
- `usn` — упрощённая декларация: базовая ставка, лимит, ставки городов
- `nds` — НДС: ставка, порог регистрации, льготные ставки
- `kpn` — КПН для ТОО на ОУР
- `ipn_ip_our` — ИПН для ИП на ОУР
- `ipn_dividends` — ИПН с дивидендов
- `samozanyaty` — режим «Самозанятый» (заменяет патент)
- `tax_deductions` — налоговые вычеты

## УСН — главный режим для нашей ЦА

### Когда применима упрощёнка

ИП на УСН платит налог с выручки. Базовая ставка установлена законом, **маслихаты могут понизить или повысить в диапазоне 2-6%**. Большинство маслихатов выбрали 2% или 3%.

**Условия применимости УСН:**
- Выручка не превышает лимит (см. `usn.annual_revenue_limit_kzt` в SoT)
- Деятельность не входит в запретительный список ОКЭД (185 кодов с 2026)
- ИП на УСН **не платит НДС**, если работает с физлицами (B2C)

### Ключевое уточнение: B2C на УСН не переходит на ОУР

Если ИП:
- На упрощённом режиме (УСН)
- Деятельность из разрешённого списка
- Клиенты — физлица (B2C)

Тогда **автоматически он не переходит на ОУР, не становится плательщиком НДС**, продолжает на УСН в пределах лимита УСН.

**Порог НДС — отдельная история.** Касается тех, кто работает с ЮЛ на ОУР. К нашей основной ЦА (B2C-микробизнес) — не применяется.

### Ставки УСН по городам

Полная таблица — в SoT yaml, поле `usn.municipality_rates`. Все города проверены через официальные решения маслихатов (`verification_status: confirmed`).

**В Wiki-обзорах и QC-отчётах** ставка УСН берётся из SoT по `city_id` клиента — никаких хардкодов в коде или в обзорах.

## Самозанятый — заменяет отменённый патент

**Патент отменён 01.01.2026.** В Wiki, движке, расчётах, материалах — везде заменять «патент» на «Самозанятый».

Параметры режима — в SoT, поле `samozanyaty`.

### Когда ИП лучше самозанятого

- Если планируется выход за лимит самозанятого
- Если нужно работать с ЮЛ
- Если нужны наёмные сотрудники (самозанятый не может нанимать)
- Если важна репутация ИП (договоры, тендеры)

## Нагрузка работодателя на ФОТ — критическое уточнение

Реальная нагрузка зависит от налогового режима работодателя:

| Режим работодателя | Нагрузка сверх оклада | Состав |
|---|---|---|
| **ИП на УСН** (наша основная ЦА) | **11.5%** | ОПВР 3.5% + СО 5% + ООСМС 3% |
| **ИП / ТОО на ОУР** | **17.5%** | + СН 6% сверх |

Соцналог (СН 6%) **отменён для УСН** с 2026 года, остался только для ОУР. В шаблоне Quick Check PDF используется правильное число 11.5% — соответствует реальности УСН.

Из оклада дополнительно удерживается у работника:
- ИПН — 10% от налогооблагаемой базы (15% на доход свыше 8 500 МРП/год)
- ОПВ — 10% от оклада
- ВОСМС работника — 2% от оклада

Налоговый вычет работника по ИПН — **30 МРП/мес** (не путать с МЗП). Конкретное значение в `tax_deductions.standard_employee_monthly` SoT.

## ИПН для ИП

| Режим | Ставка |
|---|---|
| **ИП на УСН** | не платит ИПН отдельно — налог УСН включает всё |
| **ИП на ОУР** | прогрессивная: 10% базовая, 15% свыше 8 500 МРП/год |
| **Самозанятый** | 0% ИПН (только соцплатежи 4%) |

Дивиденды ИПН — **5% до 230 000 МРП/год**, выше — 10% (снижено с 10% в 2026).

## Внутренние коэффициенты городов (НЕ налоговые ставки)

> **Не путать с УСН-ставками.** Это коэффициент пересчёта **цен / аренды / зарплат** под город — внутренняя механика движка, не связан с налогами.

| Город | Коэффициент |
|---|---|
| Алматы / Астана | ×1.05 |
| Атырау | ×1.03 |
| Актобе (база) | **×1.00** |
| Шымкент / Павлодар / Костанай | ×0.97 |
| Семей / Тараз / Петропавловск / Кызылорда | ×0.95 |

Применяются в `engine.py` к ценам/аренде/ФОТ. **В Wiki не публикуются** (внутренний параметр калькулятора, не клиентский).

## Налоги по типам бизнеса (методология)

### Розничная торговля, услуги (наша основная ЦА)
- УСН (ставка по городу из SoT)
- Без НДС, если B2C

### Общепит (рестораны, кафе, доннерные)
- УСН
- Без НДС, если B2C
- Если работа с алкоголем — отдельная лицензия + акцизы

### Услуги (барбер, маникюр, массаж, спа)
- УСН
- Без НДС
- Если мастер на проценте без оклада — оформление через ИП-договор подряда (форма самозанятости)

### Образование (Академия, языковые школы, репетиторы)
- УСН
- Если частная школа с лицензией — отдельные правила
- Возможны льготы по социальным программам (для подачи на гранты)

## Что меняется при достижении порогов

- **Превысил лимит УСН** → переход на ОУР, обязательная регистрация по НДС
- **Нанял первого работника** → возникают обязательства по ОПВР, СО, ООСМС (для УСН) или + СН (для ОУР)
- **Начал работать с ЮЛ на ОУР с НДС** → может потребоваться добровольная регистрация по НДС, если контрагент требует НДС-счёт-фактуру

## Правила использования налогов в продуктах ZEREK

- **Wiki-обзоры** — общая методология: «УСН по городу из SoT», «при найме сотрудника — нагрузка 11.5% (УСН) или 17.5% (ОУР)». Конкретные числа не пишем.
- **Quick Check** — точный расчёт, читает значения из SoT по city_id + типу клиентов
- **FinModel** — полный налоговый блок с разбивкой по месяцам, источник — SoT
- **BizPlan** — налоговый раздел в банковском формате с обоснованием режима

## Источники для актуализации

При изменении налогов (обычно 1 раз в год — с января) обновлять **только** SoT yaml:

```
data/external/kz_tax_constants_2026.yaml
```

После обновления — все слои (knowledge, docs/context, Wiki, движок) автоматически работают с новыми значениями. Не править значения в коде или в текстовых файлах вручную.

**Авторитетные источники для самой проверки:**
- adilet.zan.kz — решения маслихатов, законы РК
- НК РК (Налоговый кодекс Республики Казахстан) — текущая редакция
- mybuh.kz, uchet.kz, pro1c.kz — практические сводки
- БНС РК — статистика

## Что НЕ забывать

- Все ставки **меняются ежегодно**. МРП — каждый год новый. Не хардкодить «вечные значения».
- УСН в городах **бывает разная** (2% или 3%) — брать всегда из SoT по конкретному городу.
- НДС с физлицами **не платится** — не путать массового клиента с большим B2B.
- При найме первого работника — нагрузка **возрастает**. Закладывать в стартовый бюджет.
- Самозанятый ≠ ИП. Это разные правовые статусы с разными ограничениями.
