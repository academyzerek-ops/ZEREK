# docs/context — рабочий контекст ZEREK

> **Назначение этой папки:** хранить полное описание проекта по темам, чтобы любой Claude (Chat / Cowork / Code) загружал нужный документ под задачу — а не один файл-монстр, который всё равно никто не читает целиком.

---

## Как читать

`/CLAUDE.md` в корне репо — короткий навигатор. Содержит: текущая фаза проекта, ключевые правила, ссылки на эту папку. Читается первым в любой сессии.

Эта папка — глубина по темам. Каждый файл самодостаточен: можно дать Claude один файл — и он будет работать в этой теме корректно.

## Источники истины (Source of Truth)

Конкретные числа (ставки налогов, статусы ниш, цены, параметры архетипов) живут в YAML / data-файлах, а не в этой папке. В `docs/context/` — методология и ссылки.

| Тема | Source of Truth |
|---|---|
| Налоги КЗ 2026 | `data/external/kz_tax_constants_2026.yaml` |
| Ниши и форматы | `data/kz/niches_registry.yaml` |
| Цены продуктов | `data/products.yaml` |
| Архитектура движка | `docs/ARCHITECTURE.md` |
| Архетипы (A1 и др.) | `knowledge/archetypes/*.md` |

Полный протокол изменений и запреты на дублирование — в `10_OPERATIONS.md` (раздел «Source of Truth и протокол изменений»).

## Правила поддержания

1. **Документ обновляется при принятии решения** — а не «когда-нибудь потом».
2. **Файл с датой последнего обновления вверху** — чтобы видеть протухло или нет.
3. **Если решение отменяет старое — старое удаляется, не оставляется «для истории»**. Этот файл — живой снимок, не архив.
4. **Не дублировать**: если факт принадлежит другой теме — там и хранится, здесь только ссылка.
5. **Конкретные числа — в `data/`**, не в `docs/context/`. В `docs/context/` — методология (см. таблицу SoT выше).
6. **Размер каждого документа: 200–500 строк максимум**. Если больше — сигнал, что тема развалилась на подтемы.
