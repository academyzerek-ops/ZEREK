## Главный принцип: ниша = реалия рынка КЗ

Ниши **не подгоняются под красивое число**. Не «50 ниш», не «100 ниш» — а столько, сколько реально существует на рынке Казахстана. Счёт идёт на сотни в перспективе.

**Правила:**
- Если в реальности у владельца 2 модели бизнеса (пассив/актив, аренда/найм) → внутри ниши **2 расчёта**
- Если ниша гибридная (BARBER: и мастер, и владелец) → внутри ниши **все варианты**
- Не дробить ради раздутия счёта
- Не сокращать ради совпадения с числом
- Цель: **правдивая картина**, а не «50 готовых ниш»

**Барьер unlock массовых продаж:** 50 ниш с полной калибровкой + PDF-отчётом консультантского уровня. До этого числа — массовые продажи заблокированы (но Wiki-обзоры публикуются и работают как воронка).

## Архитектура данных по нише

3 слоя, разделение ответственности:

```
data/kz/niches_registry.yaml          ← мета-информация по нише
data/niches/<CODE>_data.yaml          ← БИЗНЕС-ДАННЫЕ, ИСТОЧНИК ИСТИНЫ
```

Лоадер сливает md + yaml в единую структуру. Excel-файл — кэш для скорости загрузки на Railway, не источник истины.

## 6 архетипов (типов экономики)

Каждая ниша принадлежит одному архетипу. Архетип определяет **метрику расчёта** и **структуру финмодели**.

| Архетип | Метрика | Примеры ниш |
|---|---|---|
| **A — Услуги с мастерами** | мастер × загрузка × чек × дней | BARBER, MANICURE, BROW, MASSAGE, COSMETOLOGY, BEAUTY, DENTAL, OPTICS, TAILOR, REPAIR_PHONE, PHOTO, DETAILING |
| **B — F&B** | чеков × средний чек × food cost % | COFFEE, BAKERY, CONFECTION, DONER, FASTFOOD, PIZZA, SUSHI, BUBBLETEA, CANTEEN, SEMIFOOD, CATERING |
| **C — Retail** | оборот запасов × наценка × чеков | MEATSHOP, FRUITSVEGS, FLOWERS, AUTOPARTS, BUILDMAT, PETSHOP, PHARMACY, GROCERY, WATERPLANT |
| **D — Абонементы** | активная база × ARPU × churn | FITNESS, YOGA, LANGUAGES, KIDSCENTER, KINDERGARTEN, COMPCLUB, ACCOUNTING (B2B-подписка) |
| **E — Проектный** | проектов × чек × маржа | REALTOR, FURNITURE, PRINTING, CARGO |
| **F — Мощность / пропускная** | постов × заполняемость × чек | AUTOSERVICE, TIRESERVICE, CARWASH, CLEAN, CARPETCLEAN, DRYCLEAN, LAUNDRY, PVZ, DRIVING, HOTEL |

Гибридные ниши (например WATERPLANT — retail + производство, CATERING — F&B + проектный, CARGO — проектный + мощность) — обрабатываются с пометкой и комбинированной моделью.

## Архетип A1 — beauty-мастер (универсальный)

Это **подархетип A**, специфичный для beauty-индустрии (MANICURE, BROW, SUGARING, MASSAGE, COSMETOLOGY, BEAUTY, BARBER-мастер). Универсально переиспользуется между нишами.

3 уровня опыта мастера меняют **одновременно** производительность + чек + CAPEX:

| Уровень | Минут/клиент | Пик/день | Среднее/день | Чек | CAPEX обучения |
|---|---|---|---|---|---|
| Нет опыта | 90 | 4 | 3 | ×0.85 | 150 000 ₸ |
| Средний | 70 | 5 | 4.5 | ×1.0 | 40 000 ₸ |
| Опытный | 50 | 7 | 5.5 | ×1.20 | 20 000 ₸ |

**«Пик» — максимум в день, не каждый день бывает.** Среднее — то, что реально складывается в месяц.

Параметры лежат в YAML архетипа A1, переиспользуются между нишами.

## Beauty-архитектура форматов (актуальный список)

Каждая beauty-ниша имеет свой набор форматов. Не все форматы есть у всех ниш — это сознательное решение, продиктованное реальностью рынка КЗ.

| Ниша | Доступные форматы |
|---|---|
| **MASSAGE** | HOME / RENT_FITNESS / OWN_OFFICE / EMPLOYED_SPA |
| **COSMETOLOGY** | STANDARD-сам / PREMIUM-владелец-найм |
| **BROW** | HOME / STUDIO simple / STUDIO nice / SALON_RENT (без ТЦ) |
| **SUGARING** | HOME / STUDIO simple / STUDIO nice (без салона, без ТЦ) |
| **BARBER** | мастер: STUDIO simple / STUDIO nice / mall / SALON_RENT — владелец: SHOP_RENT_OUT / SHOP_EMPLOY (без HOME) |
| **BEAUTY** (полноформатный салон) | PASSIVE × ACTIVE — × suburb / center / mall |

**Аренда в ТЦ:**
- Стойка / маникюрный остров: 30–70K ₸/мес
- Бутик: 5–10K ₸ за м²

**Порядок калибровки beauty:** BROW → SUGARING → MASSAGE → COSMETOLOGY → BARBER → BEAUTY.

## Маркетинг — 3-я ось анкеты QC (универсально)

Применяется ко всем нишам, не только beauty:

| Стратегия | Бюджет к базе | Выход на mature pocket |
|---|---|---|
| Консервативная | ×0.20 ramp-up | М11 |
| Средняя | ×1.0 ramp-up | М4 |
| Агрессивная | ×1.40 ramp-up | М3 |

**Антипаттерн (предупреждение в QC):** новичок + агрессивная маркетинговая стратегия = деньги впустую (мастер физически не примет столько лидов, лиды испортятся, репутация ухудшится). Новичок дома × средняя стратегия = реалистичные **140–180K ₸ чистыми**, не «315K как в идеальном сценарии».

На стр. 3 PDF-отчёта QC — **обязательный блок-предупреждение** про антипаттерн.

## Целевая аудитория — фильтр (применять ко всем нишам)

Перед калибровкой ниши проверять формат:

1. **В найме за оклад** → НЕ наша аудитория, не калибровать
2. **В найме за процент без оклада** (форма самозанятости через ИП-договор) → наша
3. **Своё дело** (аренда / свой кабинет / дом / ТЦ) → наша

Если формат не проходит фильтр — **выкидывать из калибровки сразу, без обсуждения**. Quick Check считает экономику бизнеса/самозанятости, не зарплату работника.

## Структура insight.md (текстовая часть ниши)

Каждый insight-файл содержит:

1. **Суть бизнеса** — что это, кому продаём, чем зарабатываем
2. **Экономика и маржинальность** — где деньги, какие ограничения
3. **Стартовые вложения (CAPEX)** — главные статьи
4. **Юнит-экономика** — чек, частота, объёмы
5. **Сезонность** (если есть) — с математикой кассового разрыва
6. **Кадры, команда** — типичная структура, мастер vs бренд
7. **Конкуренция** — таблицы по 22 городам КЗ, источники 2GIS
8. **Налоги по нише** (ставки, особенности)
9. **3 типа кейсов** — Хороший старт / Типичная ошибка / На грани
10. **Риски** — бренд-бизнес vs ремесло, кадры, сезонность
11. **5 наставлений** (см. `05_QC_METHODOLOGY.md`)

## Стандарты содержания insight-файлов

- Без валют, городов, годов, имён — это идея, не история конкретного человека
- Без «успешного успеха» — реалистичный тон
- Метафоры, фреймворки, концепции — главное; академический стиль и непроверенные кейсы — выкидываем
- Источники: видеоразборы практиков на YouTube (через NotebookLM), консалтинговый опыт Адиля, открытые отчёты (БНС, аналитика)
- Если кейс реальный — даём YouTube-ссылку. Если нет — помечаем «иллюстративный пример»

## Принципы калибровки новой ниши (порядок работы)

1. **Insight.md** — собирается из 3+ видеоразборов / отчётов, через NotebookLM или вручную
2. **Архетип** — определяется (A/B/C/D/E/F или подархетип)
3. **Список форматов** — что реально существует в КЗ-практике этой ниши
4. **Фильтр ЦА** — какие форматы проходят фильтр, какие выкидываются
5. **YAML с параметрами** — `data/niches/<CODE>_data.yaml`
6. **Excel-кэш** — `niche_formats_<CODE>.xlsx`, синхронизируется с YAML
7. **Wiki-обзор** — HTML-страница с уникальной палитрой
8. **PDF-отчёт QC** — тестируется на этой нише через WeasyPrint
9. **Production-ready флаг** — в `niches_registry.yaml`

## Что НЕ ниша (выкидываем)

- Найм за оклад без бизнес-намерений
- Корпорации, средний бизнес 50+ человек
- IT-стартапы (если без оффлайн-составляющей) — это для ветки Кит в Академии
- Спекулятивные сделки, перепродажа без экономической добавленной стоимости
- «Бизнес для всего мира» — мы строим под КЗ; если ниша не существует в КЗ или существует в виде 5 точек на всю страну — не делаем
