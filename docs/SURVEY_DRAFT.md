# ZEREK — Адаптивные анкеты v2: драфт 3 архетипов

Документ описывает структуру анкет для **Quick Check (express-оценка)** и
**финмодели**. Сначала универсальные вопросы → потом архетипные → потом
ниша-специфичные. Цель: 8-12 вопросов в express, 18-25 в финмодели — без
лишнего и без пробелов.

> Это **драфт для согласования**. После «одобрено» — расширяю на остальные
> 5 архетипов и оборачиваю в `09_surveys.xlsx` + рендер в `quick-check-v2.html`.

## 0. Универсальный слой (для всех 58 ниш)

| qid | Вопрос | Тип | Express | Finmodel |
|-----|--------|-----|---------|----------|
| `U_CITY` | Город (15 городов РК) | select | ✓ | ✓ |
| `U_NICHE` | Ниша | select | ✓ | ✓ |
| `U_CAPITAL` | Стартовый капитал (₸) | range slider | ✓ | ✓ |
| `U_START_MONTH` | Месяц старта | select 1-12 | (опц) | ✓ |

| `U_CREDIT` | Берёте кредит? | yes/no toggle | — | ✓ |
| `U_CREDIT_AMOUNT` | Сумма кредита (₸) | range, only if U_CREDIT=yes | — | ✓ |

| `U_CREDIT_TERM` | Срок (мес) | range 12-60 | — | ✓ |

**Логика:** в express спрашиваем минимум — город, ниша, капитал.
Финмодель добавляет налоги/кредит/рост.

## Архетип A — БЬЮТИ-ОДИНОЧКА

**Ниши:** BARBER, MANICURE, BROW, LASH, SUGARING, MASSAGE, COSMETOLOGY, BEAUTY (8 шт)

**Логика расчёта:**
выручка = (клиентов/день × раб.дней × средний чек) × кол-во кресел × загрузка%;
COGS = 5-15% от чека (расходники); FOT = % от выручки мастера или фикс ставка;
аренда зависит от формата (на дому → 0; кабинет → 80-200K ₸; салон → 200-500K).

### Express (9 вопросов)

| qid | Вопрос | Влияние |
|-----|--------|---------|
| U_CITY | Город | net rate, аренда, конкуренты |
| U_NICHE | Ниша | базовые ставки FOT, средний чек |
| `A_FORMAT` | Формат: на дому / свой кабинет / аренда кресла / салон 3+ кресел | CAPEX, аренда, кол-во кресел |
| `A_LOC` | Локация (только если формат не «на дому») — улица / ТЦ / ЖК | аренда, трафик |
| `A_AREA` | Площадь (м²) — если не home | аренда |
| `A_CHAIRS` | Сколько рабочих мест: 1 / 2 / 3-5 / 6-10 / 10+ | штат, выручка |
| `A_STAFF_MODE` | Сами работаете или нанимаете? | FOT |
| `A_CHECK` | Средний чек (₸) | выручка |
| U_CAPITAL | Капитал | проверка достаточности |

### Finmodel (доп. 9 вопросов = всего 18)

| qid | Вопрос | Влияние |
|-----|--------|---------|
| `A_LOAD` | Загрузка кресла: клиентов/день в среднем (1-8) | выручка |
| `A_COGS` | Доля расходников от чека: 5 / 10 / 15 / 20 % | себестоимость |
| `A_MARKETING` | Маркетинг: ₸/мес или % от выручки | OPEX |
| `A_FOT_MASTER` | Оплата мастера: % от чека (40/50/60) или фикс ₸/мес | FOT |
| `A_RENT` | Аренда: фикс ₸/мес или % от выручки | OPEX |
| U_ENTITY, U_CREDIT (+amount/rate/term), U_GROWTH_Y1 | универсальные | расчёт |

### Ниша-специфичные (1-2 в каждой)

| Ниша | qid | Вопрос |
|------|-----|--------|
| BARBER | `B_BARBER_TYPE` | Формат: мужской / унисекс / эконом-парикмахерская |
| MANICURE | `B_MANIC_SPEC` | Спец: базовый / наращивание / педикюр / всё |
| COSMETOLOGY | `B_MED_LIC` | Мед.лицензия: есть / планирую / нет (mandatory_if_invasive) |

**Итого express:** 9 (универс+архетип) + 0-1 (ниша) = **9-10 вопросов**

## Архетип B — ОБЩЕПИТ (кафе/доставка)

**Ниши:** COFFEE, PIZZA, SUSHI, FASTFOOD, DONER, BUBBLETEA, CANTEEN, BAKERY (8 шт)

**Логика расчёта:**
выручка = (трафик/день × конверсия × средний чек × раб.дней) + доставка;
food cost 28-35% (CDS standardno); FOT — команда (повар + кассир + помощник);
аренда сильно зависит от локации (ТЦ ≈ 5-12K ₸/м², улица ≈ 3-8K, ЖК ≈ 4-7K);
доставка — комиссия агрегаторов 15-30% от чека.

### Express (11 вопросов)

| qid | Вопрос | Влияние |
|-----|--------|---------|
| U_CITY | Город | аренда, чеки, трафик |
| U_NICHE | Ниша | food cost базовый, средний чек |
| `O_FORMAT` | Формат кухни: только навынос / с залом / полная кухня + ресторан | CAPEX, штат |
| `O_LOC` | Локация: ТЦ / улица / ЖК / БЦ | аренда, трафик |
| `O_AREA` | Площадь (м²) | аренда |
| `O_CHECK` | Средний чек (₸) | выручка |
| `O_TRAFFIC` | Трафик/день (клиентов): 30 / 60 / 100 / 200+ | выручка |
| `O_DELIVERY_SHARE` | Доля доставки: 0 / 20 / 40 / 60 % | агрегаторская комиссия |
| `O_STAFF_COUNT` | Команда (повар+кассир+...): 2 / 3-4 / 5-7 / 8+ | FOT |
| U_CAPITAL | Капитал | достаточность |

### Finmodel (доп. 11 = всего 22)

## Архетип C — B2B-ПРОФУСЛУГИ

**Ниши:** NOTARY, EVALUATION, ACCOUNTING, REALTOR (4 шт)

**Логика расчёта:**
нет трафика/площади (если есть офис — он маленький и не влияет на выручку);
выручка = клиентов/мес × средний чек ИЛИ ставка/час × оплачиваемых часов;
COGS ~ 0; FOT = специалист (часто сам владелец) + помощник; аренда офиса фикс
50-200K ₸/мес. Главные параметры — лицензия/аккредитация и средний чек.

### Express (8 вопросов)

| qid | Вопрос | Влияние |
|-----|--------|---------|
| U_CITY | Город | net rate, аренда, спрос |
| U_NICHE | Ниша | базовые ставки |
| `P_FORMAT` | Формат: соло из дома / соло-офис / агентство | CAPEX, аренда |
| `P_LICENSE` | Лицензия/аккредитация (Q_NOT_LIC / Q_IPO / Q_BANK_ACCR) | go/no-go |
| `P_STAFF` | Сами работаете или нанимаете специалистов? | FOT |
| `P_CHECK_OR_RATE` | Средний чек за услугу (₸) **или** ставка ₸/час | выручка |
| `P_CLIENTS_PER_MONTH` | Сколько клиентов/услуг в месяц планируете | выручка |
| U_CAPITAL | Капитал | достаточность |

### Finmodel (доп. 7 = всего 15)

| qid | Вопрос | Влияние |
|-----|--------|---------|
| `P_RENT` | Аренда офиса (₸/мес), 0 если из дома | OPEX |
| `P_MARKETING` | Маркетинг (₸/мес) — реклама, SEO, контекст | OPEX |
| `P_FOT_SPEC` | ФОТ специалиста (₸/мес), если не сам | FOT |
| `P_REPEAT_RATE` | Доля повторных клиентов (%) | прогноз удержания |
| U_ENTITY, U_CREDIT (+detail), U_GROWTH_Y1 | универсальные | — |

### Ниша-специфичные

| Ниша | qid | Вопрос |
|------|-----|--------|
| NOTARY | — (Q_NOT_LIC уже в P_LICENSE) | хватает |
| EVALUATION | `B_EVAL_TYPES` | Виды: недвижимость / транспорт / бизнес / всё |
| ACCOUNTING | `B_ACC_PACKAGE` | Тариф: «нулевой» / «1С» / «полный аутсорс» |
| REALTOR | `B_REAL_SEGMENT` | Сегмент: жильё / коммерция / аренда / новостройки |

**Итого express:** 8 + 0-1 = **8-9**
**Итого finmodel:** 15 + 0-1 = **15-16**

## Что дальше — после твоего «ок»

1. Аналогично распишу архетипы D-H (≈30 минут).
2. Соберу `data/kz/09_surveys.xlsx` с 3 листами:
   - «Вопросы» (qid, вопрос, тип, варианты, дефолт)
   - «Применимость» (qid → niche_id, tier=express|finmodel)
   - «Зависимости» (qid → условие показа, например «показать B_MED_LIC только если COSMETOLOGY и P_LICENSE=invasive»)
3. Engine: `get_niche_survey(niche_id, tier)` возвращает упорядоченный список вопросов с учётом текущих ответов (для условных).
4. Frontend `quick-check-v2.html` подгружает анкету через `/niche-survey/{niche_id}?tier=express` и рендерит без хардкода — сейчас вопросы наполовину захардкожены, надо вытащить.
5. Финмодель — отдельный экран `quick-check-v2.html?tier=finmodel` или новый файл `finmodel-v2.html`.

**Жду:** одобрение драфта или правки к нему перед заходом в остальные архетипы и xlsx.
