# ZEREK — Адаптивные анкеты v2: драфт 3 архетипов

Документ описывает структуру анкет для **Quick Check (express-оценка)** и
**финмодели**. Сначала универсальные вопросы → потом архетипные → потом
ниша-специфичные. Цель: 8-12 вопросов в express, 18-25 в финмодели — без
лишнего и без пробелов.

> Это **драфт для согласования**. После «одобрено» — расширяю на остальные
> 5 архетипов и оборачиваю в `09_surveys.xlsx` + рендер в `quick-check-v2.html`.

## Архетип A — БЬЮТИ-ОДИНОЧКА

**Ниши:** BARBER, MANICURE, BROW, LASH, SUGARING, MASSAGE, COSMETOLOGY, BEAUTY (8 шт)

**Логика расчёта:**
выручка = (клиентов/день × раб.дней × средний чек) × кол-во кресел × загрузка%;
COGS = 5-15% от чека (расходники); FOT = % от выручки мастера или фикс ставка;
аренда зависит от формата (на дому → 0; кабинет → 80-200K ₸; салон → 200-500K).

| qid | Вопрос | Влияние |

| `A_LOC` | Локация (только если формат не «на дому») — улица / ТЦ / ЖК | аренда, трафик |
| `A_AREA` | Площадь (м²) — если не home | аренда |
| `A_CHAIRS` | Сколько рабочих мест: 1 / 2 / 3-5 / 6-10 / 10+ | штат, выручка |
| `A_STAFF_MODE` | Сами работаете или нанимаете? | FOT |
| `A_CHECK` | Средний чек (₸) | выручка |
| U_CAPITAL | Капитал | проверка достаточности |

|-----|--------|---------|

## Архетип C — B2B-ПРОФУСЛУГИ

**Ниши:** NOTARY, EVALUATION, ACCOUNTING, REALTOR (4 шт)

**Логика расчёта:**
нет трафика/площади (если есть офис — он маленький и не влияет на выручку);
выручка = клиентов/мес × средний чек ИЛИ ставка/час × оплачиваемых часов;
COGS ~ 0; FOT = специалист (часто сам владелец) + помощник; аренда офиса фикс
50-200K ₸/мес. Главные параметры — лицензия/аккредитация и средний чек.

| `P_STAFF` | Сами работаете или нанимаете специалистов? | FOT |
| `P_CHECK_OR_RATE` | Средний чек за услугу (₸) **или** ставка ₸/час | выручка |

## Структура `09_surveys.xlsx`

Будет создан скриптом `scripts/build_09_surveys.py` со следующими листами:

### Лист «Вопросы» (catalog)

| qid | question_text | input_type | options | placeholder | min | max | step | unit | help |
|-----|---------------|------------|---------|-------------|-----|-----|------|------|------|

`input_type` ∈ {`select`, `radio`, `number`, `range`, `select_remote`, `select_format`, `multi_select`, `boolean`}.
`select_remote` — варианты подгружаются с API (например `U_CITY` → `/cities`).

### Лист «Применимость»

| qid | niche_id | tier | order | required |
|-----|----------|------|-------|----------|

`niche_id = '*'` для универсальных. `tier ∈ {express, finmodel}`. `order` — по этому полю фронт сортирует.

### Лист «Зависимости» (опционально, для условного показа)

| qid | depends_on | condition | action |
|-----|------------|-----------|--------|

Например: `U_CREDIT_AMOUNT | U_CREDIT | == yes | show`.

## План кода

1. `scripts/build_09_surveys.py` — генератор xlsx из Python-источника
   правды (всех вопросов и применимости).
2. `engine.py`:
   - Загрузка `09_surveys.xlsx` в `db.surveys_questions`, `db.surveys_applic`.
   - `get_niche_survey(db, niche_id, tier='express')` — вернёт упорядоченный
     список вопросов с метаданными.
3. `main.py`:
   - `GET /niche-survey/{niche_id}?tier=express|finmodel` → JSON.
4. `quick-check-v2.html`:
   - После выбора ниши тянет `/niche-survey/{niche_id}?tier=express`.
   - Рендерит шаги по полученному списку (не по хардкоду).
   - На confirm-странице — кнопка «Развернуть в финмодель» → подгружает
     `?tier=finmodel`, добавляет недостающие шаги, после ответа POST
     `/finmodel`.
