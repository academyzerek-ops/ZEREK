# ZEREK Knowledge Base

Эта папка — **единый источник истины** для бизнес-данных ZEREK. Здесь живут параметры ниш, регионов, налогов и архетипов. Меняешь цифру здесь — через 30 минут она работает в проде.

## Что лежит где

```
knowledge/
├── archetypes/    — универсальные параметры архетипов (A1, A2, ...)
│   └── A1_BEAUTY_SOLO.md
├── niches/        — данные по нишам
│   ├── MANICURE.md
│   ├── BARBER.md
│   └── ...
├── regions/       — города + области КЗ
│   ├── astana.md
│   ├── almaty.md
│   └── ...
├── taxes/         — налоговые параметры по годам
│   └── KZ_2026.md
├── changelog/     — большие изменения, ТЗ, решения
├── _schemas/      — JSON Schema для валидации (не трогать)
├── _templates/    — болванки для новых ниш / регионов
│
├── kz/, ru/, common/  — RAG-инсайты для PDF (старая папка, отдельный
│                         от R12.6 контур; редактируется через инсайты,
│                         не через эту базу знаний)
```

## Как редактировать

Каждый файл состоит из двух частей:

### 1. Frontmatter (YAML сверху файла)

Это **машиночитаемая** часть. Числа и параметры которые читает движок.

```yaml
---
id: astana
name_rus: Астана
avg_salary_2025: 659610
check_coef: 1.05
---
```

Если изменить `avg_salary_2025: 700000` — следующий Quick Check в Астане использует новое значение.

В Obsidian Frontmatter отображается как табличка **«Properties»** в верхней части документа. Можно редактировать через UI или через текст.

### 2. Markdown ниже

**Человекочитаемая часть.** Описание, заметки, наблюдения. Идёт в PDF info_blocks для генерации текстов.

## Как добавить новую нишу (например BARBER)

1. Скопируй `_templates/niche_template.md` → `niches/BARBER.md`.
2. Заполни frontmatter (форматы, чеки, аренда, риски).
3. В `config/niches.yaml` отметь `BARBER: { available: true }`.
4. Сохрани в Obsidian.
5. Через 30 минут (auto-commit Obsidian Git) изменение в репо → пуш на GitHub → Railway пересоберёт.

## Как изменить параметр в существующей нише

1. Открой `niches/MANICURE.md` в Obsidian.
2. Найди нужное поле во frontmatter (например `formats.HOME.base_check_astana`).
3. Поменяй число.
4. Описание изменения — в раздел `## Замечания` или `history:` массив.
5. Save → авто-коммит → прод.

## Правила хорошего тона

- **Не правь `_schemas/`** — это структурные ограничения для движка.
- **Не правь legacy YAML** в `data/niches/*_data.yaml` — для маникюра это уже legacy, для остальных ниш это пока ещё рабочий источник (миграция per-ниша по запросу).
- **Каждое значимое изменение** — добавляй строку в `history:` файла. Видна история «когда что меняли».
- **Сомневаешься** — спроси Кота через чат, **не правь** опасные поля (capex_items, налоговые ставки).

## Как настроить Obsidian Git (один раз)

См. `changelog/2026-04-26_obsidian_setup.md` (создаётся в R12.6 фазе 4).

Краткая суть:
1. Установи Obsidian.
2. Open vault → выбери `ZEREK/knowledge/`.
3. Установи плагин **Obsidian Git** через Community plugins.
4. Настрой auto-commit каждые 30 минут + auto-pull каждые 5 минут.
5. Готово — твои правки сами уходят в GitHub.
