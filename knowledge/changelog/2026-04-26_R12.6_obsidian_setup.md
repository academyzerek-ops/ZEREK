# Подключение Obsidian к knowledge/ (R12.6)

Инструкция Адилю как настроить Obsidian для редактирования базы знаний один раз. После настройки правки автоматически уходят в GitHub каждые 30 минут — следующий Quick Check использует обновлённые данные.

## Шаг 1 — Установить Obsidian

1. Открыть https://obsidian.md
2. Скачать macOS версию (DMG)
3. Установить как обычное приложение

## Шаг 2 — Открыть `knowledge/` как vault

1. Запустить Obsidian
2. На стартовом экране → **Open folder as vault**
3. Выбрать папку `/Users/adil/Documents/ZEREK/knowledge` (локальный путь к репо)
4. Готово — vault открыт

## Шаг 3 — Установить плагин Obsidian Git

1. Settings (⌘,) → **Community plugins**
2. Включить **Restricted mode** = OFF
3. **Browse** → найти **Obsidian Git** (автор Vinzent Zeband)
4. Install → Enable
5. После установки: Settings → **Obsidian Git** в левой панели

## Шаг 4 — Настройка Obsidian Git

В Settings → Obsidian Git ставим:

| Опция | Значение |
|---|---|
| Vault backup interval (minutes) | **30** |
| Auto pull interval (minutes) | **5** |
| Auto pull on startup | ✅ |
| Commit message on auto backup | `knowledge: автообновление {{date}}` |
| Disable on this device | ❌ |
| List changed files in commit body | ✅ |
| Git path | (auto) |
| Git author info | name=`Адиль`, email=`small.economy.kaz@gmail.com` |

## Шаг 5 — Тестовое редактирование

1. В Obsidian открой `knowledge/regions/astana.md`
2. В верхней части видишь блок **Properties** (frontmatter в табличке)
3. Поменяй `avg_salary_2025` на 700000 (тестовое значение)
4. Save (⌘S)
5. Через 30 минут проверь GitHub: https://github.com/academyzerek-ops/ZEREK/commits/main
6. Должен быть свежий коммит «knowledge: автообновление …»
7. Верни значение обратно (659610) и save — будет ещё один коммит

## Что куда писать

| Что хочешь поменять | Какой файл |
|---|---|
| Чек/аренда/CAPEX маникюра | `knowledge/niches/MANICURE.md` |
| Параметры опыта мастера / стратегии маркетинга | `knowledge/archetypes/A1_BEAUTY_SOLO.md` |
| ЗП в Алматы, население Астаны | `knowledge/regions/{город}.md` |
| Ставка УД / МРП / НДС в КЗ | `knowledge/taxes/KZ_2026.md` |
| Список ниш + флаги available | `config/niches.yaml` (это ТЕХНИЧЕСКИЙ файл, правится через PR с Котом) |
| Дефолтная сезонность для не-бьюти ниш | `config/defaults.yaml` (тех.файл) |

## Если что-то ломается

- **Не пушится в GitHub** → проверь интернет, потом Settings → Obsidian Git → **Pull** (вручную). Если конфликт — позвать Кота.
- **Вижу странные `<<<<<<< HEAD` в файле** → это конфликт двух правок, позвать Кота.
- **Properties не отображается как табличка** → Settings → Editor → **Properties in document** = `Visible`.
- **Случайно удалил что-то** → **History** в плагине Obsidian Git → **Reset all to last commit**.

## Что НЕ редактировать

- `_schemas/` — JSON Schema для валидации (структура полей)
- `_templates/` — болванки для новых ниш (трогать только когда добавляешь новую нишу)
- `*.legacy.yaml` файлы — машинные backward-compat файлы (R12.7+ удалит)
